# 英文法 4択演習サイト

印刷していた英文法4択（5文法レベル×100問＝**計500問**）を、塾のPC1台で
インタラクティブに解かせる演習Webアプリ。**LLM・API課金なし・オフライン動作**。

## 起動

```
cd 英語演習サイト/prototype
py run.py
```
→ ブラウザで http://127.0.0.1:8010 を開く（数学演習サイトの 8000 と同時に動かせる）。

初回は依存をインストール：`py -m pip install -r requirements.txt`

## 仕組み

- **データ**：問題の正本は `英語/英文法4択問題_{超初級|初級|中級|上級|最上級}100問.md`。
  - 最上級だけ元はHTMLなので `convert_saijokyu_html_to_md.py` で一度MD化済み。
  - `build_questions.py` が5つのMD →`questions.json`（id/印刷番号/文法レベル/単元/問題文/選択肢4/正解語/解説）。
  - **問題を編集したら** MDを直し、`py build_questions.py` を再実行すれば反映される。
- **詳しい解説**（一人学習用）：`explanations.json`（qid→`{why_correct, why_wrong{選択肢:理由}, point, example, example_ja}`）が
  別レイヤーの解説。`build_questions.py` が `questions.json` の各問に `rich` として統合する。
  - 実行時はオフラインのまま（解説は事前に書いて保存しておく方式）。
  - 採点後のフィードバックで「なぜ正解か／ほかの選択肢（選んだ誤答を強調）／ポイント／例文」を表示。未執筆の問は従来の1行解説にフォールバック。
  - **解説を増やすには** `explanations.json` に qid を足して（上のキー構成で）書き、`py build_questions.py` を再実行する。
  - 現状の執筆済み：超初級 1〜12（be動詞）。残りは順次追加予定。
- **出題**（`server.py`）：生徒名選択（＋ゲスト）→ 文法レベル選択 → 前回の未克服問（復習）を最初に出題 →
  未出題からランダム10問 → 即時採点＋解説常表示 → セット末に間違いを全正解まで再出題 →
  100問一周後は未克服問のみ → 全克服でレベル完了。選択肢は毎回シャッフルし、**採点は正解の語**で行う。
- **ゲーミフィケーション**：XP＝克服問の weight（文法レベル加重 ×1〜×5）の総和。
  全500問克服で最大XP 1500・最高ランク「文法マスター」。XPは単調増加（克服したら減らない）。
  ランクは8段の称号はしご（`data.RANKS`）。昇格時のみ控えめな演出。
- **進捗**（`history.py` / SQLite `history.db`）：生徒ごとに served/mastered/review を保持。ゲストは保存しない。
- **先生メニュー**（ヘッダ右）：
  - **紙の結果入力**：生徒→レベル→解いた範囲→誤答番号をチェック。残りは正解＝克服として反映（出所 `paper`）。
  - **弱点ビュー**：生徒×レベル×単元の正答率・未克服（復習）一覧・**よくある間違え方**（正解←選んだ誤答を回数順）。
    誤答時に選んだ選択肢を `miss` テーブルに集計する（紙入力は選択肢不明のため集計対象外＝誤答数のみ反映）。
  - **弱点プリントを作成**：その生徒の未克服問をA4モノクロの宿題プリント（問題編＋解答編）HTMLとして新タブで開く
    （`GET /api/homework?student=...`、`homework.py` が生成）。印刷番号付きなので、紙で解かせたあと「紙の結果入力」に戻せる。

## 操作

- 解答：選択肢クリック、または **キーボード 1〜4**。「次へ」は **Enter**。

## テスト

```
py test_flow.py
```
出題ループ・克服=1回正解・セット末の間違い直し・XP単調増加・次回冒頭復習・紙取り込み・最高ランクを検証。

## デザイン

UIは `claude` デザインスキル準拠（アイボリー地＋ニアブラックのインク・影ゼロ・角丸0px）。

## 既知メモ

- 超初級 問55 は元MDで選択肢 ア・エ がどちらも `read`（教材側の軽微な誤植）。正解 `am reading` は一意なので
  採点は正しく動くが、気になればMD側を修正のこと。
