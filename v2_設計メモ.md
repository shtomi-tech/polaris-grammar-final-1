# 英語演習サイト v2 設計メモ

## 目的

英文法の網羅リストをもとに、数学演習サイトの「公式 → 公式暗記4択 → 練習問題 → クリアチェッカー」と同様の流れで、英語文法を学習できる新バージョンを作成する。

英語版では、数学の「公式1個」に相当する単位を「文法事項1個」とする。

## 基本フロー

標準導線は以下とする。

1. 文法事項を選ぶ
2. 文法説明を見る
3. 文法事項の暗記4択を解く
4. 空所補充4択の練習問題を解く
5. クリアチェッカーで状態を確認する

ただし、運用上の自由度を優先し、説明だけ・暗記だけ・練習だけ・クリアチェッカーから直接開始も可能にする。

## 学習の最小単位

最小単位は「文法事項1個」とする。

例:

- be動詞
  - am / are / is
  - be動詞の肯定文
  - be動詞の否定文
  - be動詞の疑問文
- 現在完了形
  - 現在完了・継続
  - 現在完了・経験
  - 現在完了・完了
  - for / since
  - ever / never

## コース構造

UI上では、中学英文法と高校・大学受験英文法を分ける。

### 中学英文法コース

- 中1
- 中2
- 中3

### 高校・大学受験英文法コース

- 基礎
- 標準
- 発展
- 入試実戦

データ上は同じ文法事項データとして持ち、`course` と `level` で分類する。

## 画面上の階層

画面上は2階層にする。

```text
単元 → 文法事項
```

3階層にはしない。中項目に相当する情報は、必要に応じて文法事項名に含める。

例:

```text
時制
- 現在形・過去形
- 進行形
- 現在完了・継続
- 現在完了・経験
- 現在完了・完了
```

## 文法説明カード

文法事項1つにつき、説明カード1枚を持たせる。

説明カードは以下の5項目で固定する。

```text
一言ルール
形・公式
使う場面
例文
よくあるミス
```

アプリ内の文法説明は長文教材にせず、確認しやすい短いカードにする。

## 暗記4択

文法事項1つにつき、暗記4択を最低2問持たせる。

暗記4択は以下の2系統を基本とする。

```text
A. ルール確認型
B. 形・用法判別型
```

重要項目は3〜4問に増やしてもよいが、標準は2問。

### 暗記クリア条件

```text
暗記4択を全問正解でクリア
```

## 練習問題

練習問題は、初期バージョンでは空所補充4択のみにする。

文法事項1つにつき、空所補充4択を5問用意する。

### 練習クリア条件

```text
空所補充4択を全問正解でクリア
```

1周目で間違えた問題は、間違えた分だけ2周目・3周目に出題する。全問正解済みになった時点でクリアとする。

## 文法事項のクリア条件

文法事項のクリアは、暗記クリアと練習クリアの両方を満たした状態とする。

```text
文法事項クリア = 暗記クリア + 練習クリア
```

文法説明は「確認済み」として表示するが、クリア条件には含めない。

## クリアチェッカー

クリアチェッカーは統合型にする。

文法事項ごとに以下を表示する。

```text
説明: 未確認 / 確認済み
暗記: 未 / 済
演習: 未 / 済
完了: 未 / 済
```

文法事項一覧では、元の順番を維持し、状態バッジを表示する。

未クリア項目だけを見たい場合は、フィルタで切り替えられるようにする。

## 出題時の挙動

### 正解時

```text
その問題IDを正解済みに追加する
```

### 不正解時

```text
解説を表示
その問題IDは未正解のまま残す
次の問題へ進む
```

同じ問題を直後に解き直させる方式にはしない。

## 解答後の解説

空所補充4択の解説は、以下の4点を表示する。

```text
正解
なぜ正解か
他の選択肢がダメな理由
例文の訳
```

英語4択では、他の選択肢がなぜ不適切かを重視する。

## 復習仕様

復習は以下の両方を扱う。

```text
間違えた問題
クリア済み文法事項の間隔復習
```

### 間違えた問題

次回セットの最初に出す。

### クリア済み文法事項

間隔復習の初期ルールは以下。

```text
1日後 → 3日後 → 7日後 → 14日後
```

復習時は、同じ問題固定ではなく、同じ文法事項の別問題も使う。

## 通常演習での出題対象

通常演習では、未クリア項目を優先する。

クリア済み項目は通常演習から外し、復習モードで再出題する。

## 既存英語アプリの扱い

既存の「超初級〜最上級」の4択100問セットは、旧モードとして残す。

```text
新モード: 文法事項ベース
旧モード: 超初級〜最上級の100問セット
```

旧問題の文法事項への再分類は、初期実装では行わない。



```json

```

## 問題生成

問題は完全自動生成を前提とする。

ただし、授業中の安定性を優先し、実行時生成ではなく事前生成にする。

```text
grammar_items.json
→ generate_grammar_bank.py
→ grammar_bank.json
→ アプリで出題
```

### 問題数

文法事項1つあたり:

```text
説明カード: 1枚
暗記4択: 2問
空所補充4択: 5問
```

### 自動検査

生成後、最低限以下を検査する。

- 正解が1つだけか
- 選択肢が4つあるか
- 空所が1つだけか
- 解説に正解語が含まれているか
- 同じ選択肢が重複していないか
- 明らかに同じ語形が複数正解になっていないか

検査で落ちた問題は捨て、規定数に達するまで再生成する。

最大試行回数を超えた文法事項は「要確認」としてログ出力する。

## 実装方針

新バージョンは既存英語アプリを直接改修せず、別フォルダに作る。

```text
英語演習サイト/v2/
```

既存の以下は残す。

```text
英語演習サイト/prototype/
```

## v2初期版の技術方針

初期v2はフロント完結で作る。

```text
HTML / CSS / JS / JSON
localStorageで進捗保存
```

既存バックエンドへの統合は後続段階で行う。

## v2初期版の成果物

最初の成果物は、動く最小プロトタイプとする。

- 文法事項一覧

## UI方針

デザインは数学演習サイトと同じ Codex 風に合わせる。

- 温かいアイボリー地
- ニアブラックの本文
- 影なし
- ハードな罫線
- 角丸控えめ、または0
- 色ではなくタイポと下線で強調
- 学習アプリとして静かで見通しのよいUI

## ホーム画面の主導線

```text
コース

```

## 文法事項ページ

文法事項ページはタブ型にする。

```text
[説明] [暗記4択] [練習問題]
```

1つの文法事項の中で、説明・暗記・演習を切り替える。
